# Presenter Mode 使用说明

## 文件

- `presenter.html`：演讲者模式工具
- 默认 PPT：`../AI分享-2026-07/ppt-final/index.html`
- 默认讲稿：`../AI分享-2026-07/ppt-final/speaker-notes.md`

## 启动方式

在项目根目录运行：

```bash
python3 -m http.server 4311
```

然后打开：

```text
http://localhost:4311/演讲工具/presenter.html
```

不要直接双击打开 `presenter.html`。直接打开会变成 `file://`，浏览器可能禁止读取 Markdown 讲稿。

## 使用流程

1. 打开 `presenter.html`。
2. 点击「打开展示窗口」。
3. 把新打开的展示窗口拖到投影屏。
4. 让展示窗口全屏。
5. 回到本机屏幕，只操作 Presenter 窗口。

## 操作

- 下一页：`→` / 空格 / PageDown
- 上一页：`←` / PageUp
- 第一页：Home
- 最后一页：End
- 黑屏：B
- 重置计时：R
- 放大讲稿字号：`+`
- 缩小讲稿字号：`-`
- 恢复默认字号：`0`

## 显示设置

顶栏提供两组演讲前设置：

- 字号：`A-` / `默认` / `A+`
- 布局：`讲稿优先` / `均衡` / `预览优先`

默认值：

- 布局：讲稿优先
- 讲稿字号：28px

字号范围：

- 最小：18px
- 最大：40px
- 每次调整：2px

设置会自动保存到浏览器 `localStorage`，刷新后仍会保留。

## 区域折叠

顶栏的区域开关可以显示 / 隐藏：

- 配置栏
- 当前页预览
- 下一页预览
- 计时区
- 页码列表

这些状态同样会自动保存。

## 宽度拖拉

左侧预览区和右侧讲稿区之间有一条竖向拖拽条。

拖动它可以临时调整预览区宽度；拖完会自动保存。再次点击 `讲稿优先` / `均衡` / `预览优先` 会回到对应预设布局。

## 当前版本能力

- 本机屏显示当前页预览
- 本机屏显示下一页预览
- 本机屏显示当前页讲稿
- 本机屏显示计时器
- 本机操作翻页，投影窗口同步翻页
- 支持黑屏
- 支持讲稿字号调节
- 支持讲稿 / 预览布局预设
- 支持区域折叠
- 支持预览栏宽度拖拉
- 支持替换 PPT HTML 路径和讲稿 Markdown 路径

## 注意

这个版本不改原 PPT 文件。它通过原 PPT 里的 `window.deck.show(n)` 控制页码。

如果换成其他 HTML PPT，只要它也有类似的页码控制接口，适配成本很低；如果没有，也可以退化为改 URL hash 或模拟方向键。
