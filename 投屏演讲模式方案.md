# 投屏演讲模式方案

## 目标

投屏时观众只看到 HTML PPT；演讲者自己的电脑屏幕上能看到讲稿备注、当前页、下一页、计时器和翻页控制。

## 推荐方案：双窗口演讲者模式

核心思路：

- 投影屏：打开 `slides.html`，全屏展示给观众。
- 本机屏：打开一个“演讲者窗口”，显示备注、当前页、下一页、计时器、进度和控制按钮。
- 两个窗口之间同步页码：本机翻页时，投影窗口同步翻页；投影窗口翻页时，本机备注也同步。

技术上可以用浏览器原生能力实现：

- `window.open()`：从演讲者窗口打开展示窗口。
- `BroadcastChannel`：两个浏览器窗口同步当前页。
- `localStorage`：作为兼容兜底。
- `requestFullscreen()`：让展示窗口进入全屏。

## 明天最快可用方案

如果时间很紧，不改原 HTML PPT 的主体结构，优先做这个：

1. Mac 投屏模式选择“扩展屏幕”，不要选“镜像屏幕”。
2. 把 HTML PPT 浏览器窗口拖到投影屏。
3. 投影屏上的 PPT 进入全屏。
4. 本机屏打开讲稿 Markdown / PDF / 另一个 HTML 备注页。
5. 翻页时以 PPT 为准，讲稿按页码手动跟随。

这个方案最稳，开发量最低，但缺点是备注不会自动跟随页码。

## 更好的方案：加一个 presenter.html

在 `演讲工具/` 里后续可以做一个独立工具：

```text
演讲工具/
  presenter.html        # 演讲者窗口，已实现
  Presenter-Mode-使用说明.md
```

使用方式：

1. 打开 `presenter.html`。
2. 选择或配置 PPT 文件路径。
3. 点击“开始演讲”。
4. 工具自动打开展示窗口。
5. 把展示窗口拖到投影屏并全屏。
6. 演讲者窗口留在本机屏，看备注并控制翻页。

当前已做最小可用版本：

- 默认绑定 `AI分享-2026-07/ppt-final/index.html`
- 默认读取 `AI分享-2026-07/ppt-final/speaker-notes.md`
- 支持当前页 / 下一页预览
- 支持讲稿自动跟随页码
- 支持计时器、黑屏、键盘翻页
- 不改原 PPT 文件

## 演讲者窗口应该包含

- 当前页缩略图
- 下一页缩略图
- 当前页备注
- 总页数与当前页码
- 已用时间
- 上一页 / 下一页按钮
- 重置计时按钮
- 黑屏按钮：临时隐藏投影内容

## 备注来源

可以有两种方式：

### 方式 A：讲稿独立成 notes.md

适合你现在已经有演讲稿的情况。

格式建议：

```markdown
## Slide 1

这里写第一页讲稿。

## Slide 2

这里写第二页讲稿。
```

优点：不污染 PPT HTML，维护简单。

### 方式 B：备注写进 HTML PPT

每页 slide 内加隐藏备注：

```html
<aside class="notes">
这里写本页讲稿。
</aside>
```

优点：PPT 和讲稿绑定更紧，不容易错页。

缺点：需要改 PPT HTML。

## 我的建议

明天要讲，优先级应该是：

1. 先用“扩展屏幕 + PPT 全屏 + 本机看讲稿”的稳妥方案。
2. 如果还有半天以上时间，再做 `presenter.html` 双窗口同步。
3. 不建议临时大改现有 PPT HTML，除非已经确认 HTML 的翻页机制很简单。

## 最小开发判断

如果你的 HTML PPT 已经支持键盘翻页，演讲者模式只需要做三件事：

- 读取当前页码
- 发送翻页指令
- 根据页码显示对应讲稿

最小可用版本不需要动画预览、不需要复杂 UI，先保证：

- 投影窗口稳定全屏
- 本机窗口能看讲稿
- 左右方向键能同步翻页

## 彩排检查清单

- 投屏模式确认是“扩展屏幕”
- 投影屏只显示 PPT，不显示讲稿
- 浏览器通知、微信、飞书等弹窗关闭
- 本机屏亮度足够，字体调大
- PPT 翻页和讲稿页码能对应
- 断网情况下也能打开
- 准备一个 PDF 版 PPT 作为兜底
